.. Title

oracle.oci.oci_data_science_job_facts -- Fetches details about one or multiple Job resources in Oracle Cloud Infrastructure
+++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++

.. Collection note

.. note::
    This plugin is part of the `oracle.oci collection <https://galaxy.ansible.com/oracle/oci>`_ (version 2.56.0).

    You might already have this collection installed if you are using the ``ansible`` package.
    It is not included in ``ansible-core``.
    To check whether it is installed, run :code:`ansible-galaxy collection list`.

    To install it, use: :code:`ansible-galaxy collection install oracle.oci`.

    To use it in a playbook, specify: :code:`oracle.oci.oci_data_science_job_facts`.

.. version_added

.. versionadded:: 2.9.0 of oracle.oci

.. contents::
   :local:
   :depth: 1

.. Deprecated


Synopsis
--------

.. Description

- Fetches details about one or multiple Job resources in Oracle Cloud Infrastructure
- List jobs in the specified compartment.
- If *job_id* is specified, the details of a single Job will be returned.


.. Aliases


.. Requirements

Requirements
------------
The below requirements are needed on the host that executes this module.

- python >= 3.6
- Python SDK for Oracle Cloud Infrastructure https://oracle-cloud-infrastructure-python-sdk.readthedocs.io

Examples
--------

.. code-block:: yaml+jinja

    
    - name: Get a specific job
      oci_data_science_job_facts:
        # required
        job_id: "ocid1.job.oc1..xxxxxxEXAMPLExxxxxx"

    - name: List jobs
      oci_data_science_job_facts:
        # required
        compartment_id: "ocid1.compartment.oc1..xxxxxxEXAMPLExxxxxx"

        # optional
        project_id: "ocid1.project.oc1..xxxxxxEXAMPLExxxxxx"
        display_name: display_name_example
        lifecycle_state: CREATING
        created_by: created_by_example
        sort_order: ASC
